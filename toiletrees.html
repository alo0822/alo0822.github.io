Toiletrees

October 2015

Web UI Design

Toiletrees is a web application that locates the closest bathrooms for use.

OVERVIEW

What: A web application

Goal: Find the closest bathroom quickly and easily

Users: People who go to the bathroom frequently

Tools used: Photoshop, Illustrator, HTML&CSS, Google MDL, Ruby on Rails

My Role: Created designs, visuals, and prototype

INSPIRATION

A group of friends and I came up with this idea for a hackathon that we participated in. We thought about what kind of problems people run into during their daily lives and how we can make some tasks more convenient. I remembered how when I first came to America with my family, we always had trouble finding bathrooms to use. In Asia, there are a lot of public bathrooms to use in malls, convenience stores, parking lots, schools, office buildings, metro stations, etc. However, there aren't a lot of bathrooms in public places like the BART stations, and most are in restaurants or hotels only for customers to use. From here, we came up with the name "Toiletrees" and the tagline "Nature is calling" (mostly due to sleep deprivation).

FEATURES PLANNING

When we started talking about this idea, we came up with a lot of features that we wanted the application to have. We wanted to implement a rating system with a user base, so users can add, comment, and rate bathrooms. We could have information on bathrooms such as ADA accessibility, cleanliness, if it's customers only, and more. First, we decided to make a very basic application with a simple location search and results page with a map.

LOGO DESIGN

While creating the logo design, I wanted to incorporate the idea of nature and trees from the name and tagline with a simple, minimal feel. I could envision the style I wanted the trees to be, but I wasn't sure of the exact look so I tried out a few before deciding on the final design.
